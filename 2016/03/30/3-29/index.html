javascript的作用域链

例1：

var name = 'laruence';
function echo() {
     alert(name);//undefined
     var name = 'eve';
     alert(name);//eve
     alert(age);//脚本错误
}
echo();
JS权威指南中有一句很精辟的描述:
”JavaScript中的函数运行在它们被定义的作用域里,而不是它们被执行的作用域里.”

在JS中，作用域的概念和其他语言差不多， 在每次调用一个函数的时候 ，就会进入一个函数内的作用域，当从函数返回以后，就返回调用前的作用域.

JS的语法风格和C/C++类似, 但作用域的实现却和C/C++不同，并非用“堆栈”方式，而是使用列表，具体过程如下(ECMA262中所述):
任何执行上下文时刻的作用域, 都是由作用域链(scope chain, 后面介绍)来实现.
在一个函数被定义的时候, 会将它定义时刻的scope chain链接到这个函数对象的[[scope]]属性.
在一个函数对象被调用的时候，会创建一个活动对象(也就是一个对象), 然后对于每一个函数的形参，都命名为该活动对象的命名属性, 然后将这个活动对象做为此时的作用域链(scope chain)最前端, 并将这个函数对象的[[scope]]加入到scope chain中.

例2：

var func = function(lps, rps){
     var name = 'laruence';
     ........
}
func();
1.在执行func的定义语句的时候：

1)会创建一个这个函数对象的[[scope]]属性

2)并将这个[[scope]]属性, 链接到定义它的作用域链上(后面会详细介绍), 此时因为func定义在全局环境, 所以此时的[[scope]]只是指向全局活动对象window active object.

2.在调用func的时候,

1)会创建一个活动对象(假设为aObj, 由JS引擎预编译时刻创建, 后面会介绍)

2)并创建arguments属性, 然后会给这个对象添加俩个命名属性aObj.lps, aObj.rps;
对于每一个在这个函数中申明的局部变量和函数定义, 都作为该活动对象的同名命名属性.

3)然后将调用参数赋值给形参数，对于缺少的调用参数，赋值为undefined。

3.然后将这个活动对象做为scope chain的最前端, 并将func的[[scope]]属性所指向的,定义func时候的顶级活动对象, 加入到scope chain.

例3：

function factory() {
     var name = 'laruence';
     var intro = function(){
          alert('I am ' + name);
     }
     return intro;
}

function app(para){
     var name = para;
     var func = factory();
     func();
}

app('eve');//I am laruence
